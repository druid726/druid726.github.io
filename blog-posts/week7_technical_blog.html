What is SQL?

SQL is an acronym for Structured Query Language and is used for managing data held in a relational database management system. SQL has two components one: a data definition language and the other: a data manipulation language. The scope of SQL includes data insert, query, update and delete, schema creation and modification, and data access control. The data is stored in tables with rows much like a spreadsheet and the column titles must be set beforehand. Data is usually broken up into several different tables and depending on the data requested the tables will need to be joined for the requested data to be retrieved.

NoSQL Databases Explained

NoSQL, or 'Not Only SQL', represents the new class of data management technologies designed to meet the increasing volume, velocity, and variety of data that organizations are storing, processing, and analyzing.

Compared to relational databases, NoSQL databases are more scalable and provide superior performance. NoSQL databases address the opportunities that the relational model does not, including:

  • Large volumes of structured, semi-structured and unstructured data
  • Agile sprints, quick iteration, and frequent code pushes
  • Flexible, easy to use object-oriented programming
  • Efficient, scale-out architecture instead of expensive, monolithic architecture
  • Dynamic Schemas

SQL versus NoSQL

There really doesn't seem to be significant reason to choose SQL over NoSQL. Based on the differences listed below NoSQL appears to come out the clear winner.

NoSQL

  • NoSQL databases are more scalable
  • NoSQL databases are built to allow the insertion of data without a predefined schema
  • NoSQL databases scale horizontally, by adding servers instead of concentrating more capacity in a single server
  • NoSQL databases support automatic replication

SQL

  • SQL databases require that schemas be defined before you can add data
  • SQL databases usually scale vertically – a single server has to host the entire database to ensure reliability and continuous availability of data
  • SQL databases need to be taken offline to update
  • SQL databases require separate applications to manage replication
