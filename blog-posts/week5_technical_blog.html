Instance Variables — What are they?

Instance variables are variables inside a method, inside an object which is inside a class and are identifed by an @ at the beginning of the name. Their scope is limited to whatever object self refers to. One of the neat things about them is that two different objects, even if they belong to the same class, can have different values for their instance variables. Each instance variable's scope refers to the particular instance of that class hence the name instance variable. From outside the object, instance variables cannot be altered or even observed (i.e., ruby's instance variables are never public) except when explicitly provided by the programmer (see Accessor below).

  • Instance variables do not need to be declared
  • instance variable scope refers to the particular instance of that class
  • Instance variables have the nil value until they are initialized

Accessors

You normally cannot access instance variables from outside of the an object. This would break the rules of encapsulation. So, in order to provide access to instance variables and accessor method must be declared.

There are three of these helper methods. They must be run in the class scope inside the class but outside of any methods. Also, these accessors typically go at the top of the class. That gives the reader an instant overview of which member variables will be available outside the class.

  • attr_reader – allows you to view and use the instance variable but not change it.
  • attr_writer – allows you to change and use the instance variable. This is tricky – you really don't want instance variable changed outside the method they were written it but if you need it you have it.
  • attr_accessor – grants both _reader and _writer attributes.
