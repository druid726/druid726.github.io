Week 3 of Phase 0

31 May 2014

JavaScript is the programming language used to build advanced Web sites and applications. Its popularity is derived from the ease of use, making web pages interactive, enabling faster response times and its flexibilty allows for use with multiple paradigms (including mobile devices) and both client-side and server-side servers with third-party frameworks and libraries.

What's JavaScript Good For?

  • Livening pages up – enhancing websites and making them interactive
  • Data entry validation – it can validate the format of credit cards, phone numbers, email addresses, etc.
  • Handling cookies – storing information on the user's machine

What's JavaScript Not So Good For?

  • Security – JavaScript snippets allow outside access to the user's system
  • Search Engine Optimization – search spiders cannot follow JavaScript links so the page might not show up in a search
  • Rendering – JavaScript may render differently on different browsers resulting in inconsistencies with functionality and interfaces

There are many reasons to use JavaScript, it provides benefits to the user, the programmer and both the client and server sides. And while the downsides can be significant there are many third-party frameworks and add-ons to combat them.
