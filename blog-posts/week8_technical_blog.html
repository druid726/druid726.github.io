Dynamic Typing? Static Typing? Weak, Strong — what exactly are we talking about? Well, I do know that we are not talking about how fast you touch the keyboard but the 'type' as in data type (int, strings).

Dynamic Typing

A dynamically typed language is a language where the type of a variable can be altered at any time and does not need to be declared or initialized. (It is a string, now it is a Fixnum, now it is a Time object, etc.)

Static Typing

When we talk about static typing, the type of a variable is resolved in advance by the compiler. With statically typed languages, you cannot say that x is a string and, a few lines after, that it is an integer.

Strong Typing

Programming languages in which variables have specific data types are strong typed. This implies that in strong typed languages, variables are necessarily bound to a particular data type.

Weak Typing

As opposed to strong typed languages, weak typed languages are those in which variables are not of a specific data type. It should be noted that this does not imply that variables do not have types; it does mean that variables are not "bound" to a specific data type.

Dynamic Typing is Good

In a dynamic typed language, you don't have to initialize variables, which is a big bonus for many developers. Programmers like the fact that you can use a variable at will when required (without having to initialize it). Dynamic typing, in fact, does save you from writing a few "extra" lines of code, which, in turn, means less time spent writing code.

Or Is It?

The very characteristic of dynamic typed languages that appeals to many developers is also a pitfall, and a major one at that. Consider the following simple example:

/* Python code */
my_variable = 10
while my_variable > 0:
i = foo(my_variable)
if i < 100:
my_variable++
else
my_varaible = (my_variable + i) / 10 // spelling error intentional

As you can see in the above code, my_varaible is a spelling mistake that the programmer could have very well made. The problem here is that, since Python is dynamically typed, it will not return an error, but instead will create a new variable called my_varaible. So, now we have two variables: my_variable and my_varaible. This obviously is a serious problem; some would suggest that forced variable declaration is an important requirement in any programming language.

There are obvious advantages and disadvantages to both dynamic and static typed languages, as well as strong and weak. I guess it boils down to personal preference.
