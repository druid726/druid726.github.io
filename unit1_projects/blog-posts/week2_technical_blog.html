Displaying Inline vs Inline Block

24 May 2014

Block, Inline and Inline-Block are display properties in CSS for elements in HTML. First let's discuss what block and inline elements are and then we can discuss the hybrid of the inline-block.

Block

Block elements will display across the entire horizontal space if a width is not declared. Also block elements start on a new line by default. A block will take width and margin settings. When block elements appear stacked on top of each other their top and bottom margins will overlap – for example if the top and bottom margins are 10px the elements will display with a 10px margin instead of a 20px margin.

Some examples of block elements: address, article, aside, audio, blockquote, canvas, div, fieldset, figcaption, figure, footer, form, h1, h2, h3, h4, h5, h6, header, ol, output, p, section, table, ul and video.

An example of a block element.

Inline

Inline elements will only take up the space of content and will not accept a specified width – css will ignore it if declared. Nor are you able to set top and bottom margins but you can set the left and right. When set side by side inline left and right margins will combine unlike block elements – if the left and right margins are 5px the space will be 10px (5px + 5px = 10px) between the elements.

The following are some examples of inline elements: img, map, object, script, span, button, input, label, select and textarea.

An example of an inline element.

Inline-Block

The magical hybrid of inline-block allows elements to sit side by side and have height and width set. It also allows margins and padding to be set on all sides.

An example of an inline-block element.
